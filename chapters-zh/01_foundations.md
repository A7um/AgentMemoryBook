# 第一章：基础篇 — 什么是智能体记忆？

## 无状态问题

大语言模型在设计上是无状态的。每当你向 GPT-4o 或 Claude 发送一条消息时，模型对之前的任何交互都没有内在的记忆。它仅根据当前上下文窗口中的 token 来生成响应——上下文窗口是一个固定大小的缓冲区，通常在 8K 到 1M token 之间，具体取决于模型。

这对于一次性任务来说完全没问题："翻译这段话"、"写一首关于猫的俳句"。但当你需要一个能够长期运行的 AI 智能体时——一个能学习你偏好的个人助手、一个能记住你架构决策的编程智能体、一个了解用户历史的客服机器人——无状态就成了根本性的瓶颈。

上下文窗口不是记忆。它是短期工作空间。记忆是上下文窗口滑走*之后*仍然持久存在的东西。

## 智能体记忆到底是什么

智能体记忆是 AI 智能体**跨交互持久化、组织和选择性回忆信息**的能力。它将一个无状态的文本生成器转变为一个真正具有适应性的系统。

一个有用的类比来自 Du 等人 2026 年的综述：智能体记忆以**写入–管理–读取循环**的方式运作，与智能体的感知和行动周期紧密耦合：

```
          ┌─────────────────────────┐
          │    Agent Perception      │
          │  (new message / event)   │
          └────────────┬────────────┘
                       │
                       ▼
              ┌────────────────┐
              │   WRITE        │  Extract salient facts
              │   (Encoding)   │  from the interaction
              └───────┬────────┘
                      │
                      ▼
              ┌────────────────┐
              │   MANAGE       │  Consolidate, update,
              │   (Storage)    │  forget, reorganize
              └───────┬────────┘
                      │
                      ▼
              ┌────────────────┐
              │   READ         │  Retrieve relevant
              │   (Retrieval)  │  context for next action
              └───────┬────────┘
                      │
                      ▼
          ┌─────────────────────────┐
          │    Agent Action          │
          │  (generate response)     │
          └─────────────────────────┘
```

每一个记忆系统——从最简单的"保存最近 10 条消息"的缓冲区到最复杂的时序知识图谱——都实现了这个循环的某种版本。

## 记忆 vs. RAG vs. 上下文工程

这三个概念相关但各有不同：

**RAG（检索增强生成）** 从静态语料库中检索相关文档或片段来增强模型的提示词。它是记忆系统可以使用的一种*工具*，但它本身不是记忆。RAG 不会从对话中学习，不会更新已知信息，也不会遗忘过时的信息。

**上下文工程** 是决定哪些内容进入模型提示词的更广泛学科。它包括提示词设计、工具描述、系统指令和动态选择的上下文。记忆是上下文工程的一个输入。

**智能体记忆** 是一个*随时间从交互中学习*的系统。它提取、存储、更新和检索关于用户、智能体自身经验以及不断变化的世界状态的信息。记忆随每次交互而变化。RAG 则不会。

## 记忆分类体系

学术文献已在多种相互重叠的方式上达成共识来对记忆进行分类。以下是综合 2025–2026 年主要综述的统一视图：

### 按时间范围分类

| 类型 | 持续时间 | 示例 |
|------|----------|------|
| **工作记忆** | 当前会话 | 本次对话中的消息 |
| **短期记忆** | 最近几次会话 | 你最近 5 次对话的摘要 |
| **长期记忆** | 无限期 | "用户偏好 TypeScript 而非 JavaScript" |

### 按表示形式分类

| 形式 | 存储方式 | 示例 |
|------|----------|------|
| **Token 级** | 上下文窗口中的原始文本 | 对话历史、摘要 |
| **参数化** | 嵌入模型权重中 | 微调的偏好（实践中较少见） |
| **结构化** | 外部数据库 | 向量存储、知识图谱、文件 |
| **隐式** | 隐藏状态 / KV 缓存 | 实验性的；尚未达到生产就绪 |

### 按功能分类（存储什么）

| 功能 | 内容 | 示例系统 |
|------|------|----------|
| **事实记忆** | 关于世界的客观事实 | "Alice 在 Google 工作" |
| **经验记忆** | 智能体自身的过去行为 | "我向 Bob 推荐了 Python" |
| **工作记忆** | 当前任务状态 | 活跃的对话 + 检索到的上下文 |
| **偏好记忆** | 用户的喜好和厌恶 | "用户偏好简洁的回答" |
| **程序性记忆** | 如何执行操作 | 学到的任务模式、技能 |

### 按存储架构分类

| 架构 | 描述 | 权衡 |
|------|------|------|
| **缓冲区 / 线性** | 固定大小的消息队列 | 简单，但会丢失旧上下文 |
| **向量存储** | 嵌入 + 相似度搜索 | 语义召回好，关系处理差 |
| **知识图谱** | 实体 + 类型化关系 | 关系丰富，构建成本高 |
| **层级树** | 领域 → 主题 → 子主题 | 有组织，适合浏览 |
| **文件系统** | 文件和目录 | 人类可读，工具友好 |
| **混合** | 以上多种方式的组合 | 准确度最高，复杂度也最高 |

## 六大核心操作

根据 Du 等人 2025 年的操作综述，每个记忆系统都实现了以下六种基本操作的某个子集：

1. **固化** — 将原始交互转换为存储的记忆（提取）
2. **索引** — 组织记忆以便未来高效检索（嵌入、图构建）
3. **检索** — 在需要时找到正确的记忆（搜索、排序）
4. **更新** — 当新信息到达时修改已有记忆（冲突解决）
5. **遗忘** — 移除或降低过时、无关或矛盾信息的优先级
6. **压缩** — 对记忆进行摘要或压缩以适应 token 预算

记忆系统的精密程度很大程度上取决于它如何处理操作 4（更新）、5（遗忘）和 6（压缩）。任何系统都能存储和检索事实。但很少有系统能在已存储"用户住在纽约"的情况下，优雅地处理"用户搬到了伦敦"这一信息。

## 为什么记忆很难

AI 智能体的记忆看似简单实则困难重重，因为存在多个叠加的挑战：

**1. 更新问题**
当用户说"我刚搬到伦敦"时，系统必须找到旧事实（"住在纽约"），识别矛盾，并解决它——同时不能丢失用户*曾经*住在纽约的历史记录。

**2. 相关性问题**
并非用户说的每句话都值得记住。"你能把这个翻译成法语吗？"不是一个值得记忆的事实。"我在学法语，因为我们要搬到巴黎了"则是。区分信号和噪声需要判断力。

**3. Token 预算**
上下文窗口是有限的。即使有 1M token 的窗口，你也不能注入用户创建过的每一条记忆。记忆系统必须选择最相关的子集，并在 token 预算内呈现。

**4. 延迟约束**
用户期望在 2 秒内获得响应。记忆检索必须快速——通常在 100 毫秒以内——这排除了对每个查询都进行昂贵的多跳图遍历。

**5. 隐私边界**
记忆包含高度个人化的信息。系统必须处理数据隔离（用户 A 的记忆绝不能泄露到用户 B 的上下文中）、加密、保留策略以及被遗忘权。

**6. 多智能体问题**
当多个智能体共享记忆时——一个编程助手和一个个人助手同时服务同一用户——它们需要对同一记忆存储的协调但有作用域限制的访问。

## 2026 年的格局

智能体记忆领域已经爆发式增长。2023 年，MemGPT 引入了"记忆即操作系统"的范式，当时基本上是独树一帜。到 2026 年初，已经有：

- **15+ 个专用记忆框架**（Mem0、Hindsight、OpenViking、Zep、Cognee、Letta、ByteRover、Supermemory、RetainDB、Honcho 等）
- **每家主要 AI 实验室**都在推出记忆功能（OpenAI ChatGPT Memory、Claude Memory、Gemini Personal Intelligence）
- **4+ 篇学术综述**提供分类体系和基准测试
- **活跃的基准测试竞争**，在 LongMemEval 和 LoCoMo 上的分数在 18 个月内从 60% 上升到 96%

这个领域正在从"我们能让智能体记住东西吗？"转向"我们如何让它们记得*好*、大规模地记、保护隐私地记，并且不基于过时信息产生幻觉？"

本书的其余部分将深入探讨这个问题。
