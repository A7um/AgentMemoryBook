# 第六章：决策指南 — 选择合适的记忆系统

你已经了解了架构、技术和基准测试。现在的问题是：你到底应该使用哪个系统？本章为不同场景提供决策框架。

---

## 第一个问题：你需要哪种记忆？

| 记忆类别 | 含义 | 示例系统 |
|-------------|---------------|-----------------|
| **个性化** | 记住每个用户的偏好和历史 | Mem0, RetainDB, Honcho |
| **机构知识** | 从文档和数据中构建领域专业知识 | Cognee, Zep/Graphiti |
| **智能体学习** | 智能体从自身经验中改进 | Hindsight, Letta, OpenViking |
| **统一上下文** | 整体管理记忆 + 知识 + 技能 | OpenViking, ByteRover, Supermemory |

大多数应用需要某种组合。一个客服机器人既需要个性化（记住这个用户），也需要机构知识（了解产品）。一个编程智能体既需要智能体学习（记住架构决策），也需要统一上下文（项目文件 + 记忆）。

---

## 决策树

```
开始：你在构建什么？
│

│   ├─► 想要开源 + 经过验证？→ Mem0（最大社区）

│   ├─► 本地优先，基于文件？→ ByteRover（Context Tree）

│   └─► 跨工具记忆？→ OpenViking（文件系统范式）
│
├─► 企业知识系统
│   │
│   ├─► 时序推理至关重要？→ Zep/Graphiti（双时态知识图谱）
│   ├─► 多源文档接入？→ Cognee（38+ 连接器）
│   └─► 一体化平台？→ Supermemory（记忆 + RAG + 连接器）
│
├─► 长周期自主智能体
│   │
```

## 基于场景的推荐

### 场景 1："我需要我的聊天机器人记住用户"

**最佳选择：** Mem0、RetainDB 或 Honcho

| 如果你… | 选择 | 原因 |
|-----------|--------|-----|
| 想要最经受实战检验的方案 | **Mem0** | 38K+ star，丰富文档，大型社区 |
| 想要最快的集成 | **RetainDB** | 3 行代码，30 分钟内完成设置 |
| 想要最深度的个性化 | **Honcho** | 辩证推理，自动用户建模 |
| 想要免费的自托管选项 | **Mem0** | Apache 2.0，可本地运行 |

**架构模式：**
```
用户消息 → 记忆检索 → 上下文组装 → LLM → 响应 → 记忆更新
```

### 场景 2："我正在构建一个能记住项目上下文的编程智能体"

**最佳选择：** ByteRover、Letta 或 OpenViking

| 如果你… | 选择 | 原因 |
|-----------|--------|-----|
| 想要基于文件、可审计的记忆 | **ByteRover** | Markdown 文件，类 git 版本控制 |
| 想要完整的智能体框架 | **Letta** | 智能体 + 记忆 + 部署一站式方案 |
| 想要统一上下文（记忆 + 资源 + 技能） | **OpenViking** | 文件系统范式，分层加载 |
| 想要集成 Cursor/Claude Code | **ByteRover** 或 **OpenViking** | 两者均支持 MCP |

### 场景 3："我需要具有时序推理能力的企业知识管理"

**最佳选择：** Zep/Graphiti 或 Cognee

| 如果你… | 选择 | 原因 |
|-----------|--------|-----|
| 需要"这什么时候变化的？"类查询 | **Zep/Graphiti** | 双时态知识图谱 |
| 需要从多个数据源接入 | **Cognee** | 38+ 连接器，ECL 管道 |
| 需要处理对话 + 业务数据 | **Zep** | 通过 Graphiti 处理两者 |
| 想要开源图引擎 | **Graphiti** | 独立组件，可脱离 Zep 云使用 |

### 场景 4："我想要记忆任务上的最高准确率"

**最佳选择：** Hindsight 或自建系统

基于当前基准测试：
- **LongMemEval：** Agentmemory V4（96.2%）是定制构建的；Hindsight（91.4%）是最佳框架
- **LoCoMo：** ByteRover（92.2%）领先；Hindsight（89.6%）紧随其后

如果基准准确率是你的首要标准：
1. 从 **Hindsight** 开始 — 框架中准确率最高，MIT 许可证
2. 考虑 **ByteRover** 用于 LoCoMo 类型的工作负载
3. 要达到绝对最高，自建系统（如 Agentmemory V4），采用 6 信号混合检索管道

### 场景 5："我想要一个什么都能做的一体化解决方案"

**最佳选择：** Supermemory

Supermemory 将记忆 + RAG + 用户画像 + 提取器 + 连接器打包成一个 API。代价是在任何单一领域的专业化程度较低，但便利性非常显著。

如果你需要自托管：**OpenViking** 提供了最接近的一体化开源替代方案。

## 自建 vs 购买分析

### 何时选择自建

- 你的记忆需求是领域特定的（如医疗、法律、金融）
- 你需要对管道的每个方面拥有完全控制权
- 你有 3 名以上工程师的团队可以维护系统
- 你的规模足以证明投入的合理性（百万级用户）
- 你在针对特定基准或评估指标进行优化

### 何时使用框架

- 你需要在数周而非数月内在生产环境中运行记忆系统
- 你的需求与现有框架的功能一致
- 你希望受益于社区的改进和更新
- 你没有专职的记忆基础设施工程师
- 你在构建产品，而不是构建记忆系统

### 混合方案

许多生产系统采用组合方式：
1. 使用框架作为核心管道（如 Mem0 或 Hindsight 进行提取 + 检索）
2. 针对领域特定的记忆操作使用自定义逻辑
3. 使用托管服务作为基础设施（向量数据库、图数据库）

## 成本考量

| 系统 | 免费额度 | 付费起步价 | 可自托管 |
|--------|-----------|------------------|-------------|
| Mem0 | 开源 | $249/月（Pro 含图功能） | 是 |
| OpenViking | 开源 | 免费（云部署） | 是 |
| Hindsight | 开源 | 云定价待定 | 是（Docker） |
| ByteRover | CLI 免费 | SOC 2 云（定制） | 是（本地） |
| Zep/Graphiti | Graphiti 开源 | 1K 免费额度，$25/月 | 仅 Graphiti |
| Supermemory | 1M token/月 | $19/月 Pro | 仅企业版 |
| Honcho | $100 免费额度 | $2/百万 token | 是 |
| Cognee | 开源 | $35/月 开发者版 | 是 |
| RetainDB | 10K 操作/月 | 按用量计费 | 企业版 |
| Letta | 开源 | 云定价待定 | 是 |

### 隐性成本

1. **记忆操作的 LLM API 调用费用：** 大多数系统使用 GPT-4o-mini 或类似模型进行提取/整理。大规模使用时，每月可能花费 $100–1,000+。
2. **向量数据库托管：** 如果非自托管，托管向量数据库每月增加 $50–500，视规模而定。
3. **图数据库托管：** Neo4j Aura 起步价约 $65/月。Zep 必需，Mem0g 可选。
4. **工程师时间：** 自托管方案需要持续维护。预估需要一名工程师 10–20% 的时间。

## 迁移路径

如果你已经在使用某个系统并想要切换：

| 从 | 到 | 迁移难度 |
|------|----|---------------------|
| 无记忆 → Mem0 | 简单 | 文档完善的快速入门 |
| 无记忆 → RetainDB | 非常简单 | 3 行代码 |
| Mem0 → Hindsight | 中等 | 抽象方式不同（需要将记忆类型映射到网络） |
| Supermemory → RetainDB | 简单 | RetainDB 提供迁移指南 |
| MEMORY.md 文件 → Honcho | 简单 | Honcho 自动检测并提供迁移 |
| MEMORY.md 文件 → OpenViking | 简单 | OpenViking 支持从 markdown 导入 |
| MemGPT → Letta V1 | 中等 | 架构变更（heartbeats → 原生推理） |
| 任意 → ByteRover | 简单 | `brv curate -f ~/notes/MEMORY.md` 导入现有文件 |

## 总结

**如果你刚刚起步：** 选择 Mem0（最大社区，最多文档）或 RetainDB（最快集成）。

**如果准确率至关重要：** 选择 Hindsight（框架中基准测试结果最佳）。

**如果你在构建编程智能体：** 选择 ByteRover（专为此设计）或 Letta（完整框架）。

**如果你需要企业知识管理：** 选择 Zep/Graphiti（时序推理）或 Cognee（知识图谱）。

**如果你想要一个 API 搞定一切：** 选择 Supermemory（功能覆盖最广）。

**如果个性化是你的核心产品：** 选择 Honcho（最深度的用户建模）。

**如果你想要理解基本原理：** 阅读本书，然后用 Mem0 的开源代码构建一个简单原型，亲身体验写入–管理–读取循环在实践中的运作方式。
