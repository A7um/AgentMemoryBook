# 第一章：基础篇 — 什么是智能体记忆？

## 无状态问题

大语言模型天生是无状态的。每当你向 GPT-4o 或 Claude 发出一条消息，模型并不会记住此前的任何交互。它只依赖当前上下文窗口中的 token 来生成回复——所谓上下文窗口，就是一块固定大小的缓冲区，通常在 8K 到 1M token 之间，具体取决于模型。

做一次性任务倒是没问题："翻译这段话"、"写一首关于猫的俳句"。但要打造一个能持续运行的 AI 智能体——比如一个能学习你偏好的个人助手、一个能记住架构决策的编程智能体、一个熟悉用户历史的客服机器人——无状态就成了根本瓶颈。

上下文窗口不等于记忆，它只是一块短期工作空间。真正的记忆，是上下文窗口滑走*之后*依然留存的东西。

## 智能体记忆到底是什么

所谓智能体记忆，是指 AI 智能体能够**在多次交互之间持久保存、有序组织并按需回忆信息**的能力。有了它，无状态的文本生成器才能变成一个真正会适应、会成长的系统。

Du 等人在 2026 年的综述中提出了一个很好的类比：智能体记忆以**写入–管理–读取**的循环方式运作，与智能体的感知-行动周期紧密配合：

```
          ┌─────────────────────────┐
          │    Agent Perception      │
          │  (new message / event)   │
          └────────────┬────────────┘
                       │
                       ▼
              ┌────────────────┐
              │   WRITE        │  Extract salient facts
              │   (Encoding)   │  from the interaction
              └───────┬────────┘
                      │
                      ▼
              ┌────────────────┐
              │   MANAGE       │  Consolidate, update,
              │   (Storage)    │  forget, reorganize
              └───────┬────────┘
                      │
                      ▼
              ┌────────────────┐
              │   READ         │  Retrieve relevant
              │   (Retrieval)  │  context for next action
              └───────┬────────┘
                      │
                      ▼
          ┌─────────────────────────┐
          │    Agent Action          │
          │  (generate response)     │
          └─────────────────────────┘
```

所有记忆系统——从最简单的"保存最近 10 条消息"缓冲区，到最复杂的时序知识图谱——都是这个循环的某种变体。

## 记忆 vs. RAG vs. 上下文工程

这三个概念虽然相关，但各有侧重：

**RAG（检索增强生成）** 从静态语料库中检索相关文档或片段，注入到模型的提示词中。它是记忆系统可以借助的一种*工具*，但本身并不是记忆——RAG 不会从对话中学习，不会更新已知信息，也不会遗忘过时的内容。

**上下文工程** 是一门更大的学问：决定什么内容该进入模型的提示词。它涵盖提示词设计、工具描述、系统指令以及动态上下文选取。记忆是上下文工程的输入之一。

**智能体记忆** 则是一个*在持续交互中不断学习*的系统。它能提取、存储、更新和检索关于用户、智能体自身经验以及动态变化的世界状态的信息。每一次交互都会改变记忆，而 RAG 不会。

## 记忆分类体系

学术界对记忆的分类方式虽有重叠，但已基本形成共识。下面综合 2025–2026 年主要综述，给出一个统一视图：

### 按时间范围分类

| 类型 | 持续时间 | 示例 |
|------|----------|------|
| **工作记忆** | 当前会话 | 本次对话中的消息 |
| **短期记忆** | 最近几次会话 | 你最近 5 次对话的摘要 |
| **长期记忆** | 无限期 | "用户偏好 TypeScript 而非 JavaScript" |

### 按表示形式分类

| 形式 | 存储方式 | 示例 |
|------|----------|------|
| **Token 级** | 上下文窗口中的原始文本 | 对话历史、摘要 |
| **参数化** | 嵌入模型权重中 | 微调的偏好（实践中较少见） |
| **结构化** | 外部数据库 | 向量存储、知识图谱、文件 |
| **隐式** | 隐藏状态 / KV 缓存 | 实验性的；尚未达到生产就绪 |

### 按功能分类（存储什么）

| 功能 | 内容 | 示例系统 |
|------|------|----------|
| **事实记忆** | 关于世界的客观事实 | "Alice 在 Google 工作" |
| **经验记忆** | 智能体自身的过去行为 | "我向 Bob 推荐了 Python" |
| **工作记忆** | 当前任务状态 | 活跃的对话 + 检索到的上下文 |
| **偏好记忆** | 用户的喜好和厌恶 | "用户偏好简洁的回答" |
| **程序性记忆** | 如何执行操作 | 学到的任务模式、技能 |

### 按存储架构分类

| 架构 | 描述 | 权衡 |
|------|------|------|
| **缓冲区 / 线性** | 固定大小的消息队列 | 简单，但会丢失旧上下文 |
| **向量存储** | 嵌入 + 相似度搜索 | 语义召回好，关系处理差 |
| **知识图谱** | 实体 + 类型化关系 | 关系丰富，构建成本高 |
| **层级树** | 领域 → 主题 → 子主题 | 有组织，适合浏览 |
| **文件系统** | 文件和目录 | 人类可读，工具友好 |
| **混合** | 以上多种方式的组合 | 准确度最高，复杂度也最高 |

## 六大核心操作

Du 等人在 2025 年的综述中将记忆操作归纳为六种，每个记忆系统都会实现其中的某个子集：

1. **固化** — 将原始交互转换为可存储的记忆（提取）
2. **索引** — 组织记忆以便后续高效检索（嵌入、图构建）
3. **检索** — 在需要时找到正确的记忆（搜索、排序）
4. **更新** — 当新信息到达时修改已有记忆（冲突解决）
5. **遗忘** — 移除或降低过时、无关或矛盾信息的优先级
6. **压缩** — 对记忆进行摘要或精简以适应 token 预算

一个记忆系统有多精密，很大程度上取决于它怎么处理更新（4）、遗忘（5）和压缩（6）。存和取谁都会做，难的是：系统里已经存了"用户住在纽约"，这时用户说了句"我搬到伦敦了"——怎么优雅地处理？

## 为什么记忆很难

给 AI 智能体做记忆，看起来简单，做起来难——难点在于多个挑战互相叠加：

**1. 更新问题**
当用户说"我刚搬到伦敦"，系统必须找到旧事实（"住在纽约"），识别出矛盾并加以解决——同时还不能丢失用户*曾经*住在纽约的历史记录。

**2. 相关性问题**
并非用户说的每句话都值得记住。"你能帮我把这个翻译成法语吗？"算不上什么有价值的事实；但"我在学法语，因为我们要搬到巴黎了"就很值得记下来。从噪声中筛出信号，需要判断力。

**3. Token 预算**
上下文窗口终归有限。哪怕窗口有 1M token，也不可能把用户产生过的每一条记忆都塞进去。记忆系统必须精选出最相关的子集，在 token 预算内呈现。

**4. 延迟约束**
用户期望在 2 秒内得到响应。记忆检索必须够快——通常要在 100 毫秒以内——这意味着不可能对每个查询都做昂贵的多跳图遍历。

**5. 隐私边界**
记忆中存着大量私密信息。系统必须做好数据隔离（用户 A 的记忆绝不能出现在用户 B 的上下文中）、加密、保留策略，以及用户的被遗忘权。

**6. 多智能体问题**
当多个智能体共享记忆——比如一个编程助手和一个生活助手同时服务同一用户——它们就需要在同一记忆存储上做协调访问，同时严格限定各自的可见范围。

## 2026 年的格局

智能体记忆赛道已经迎来爆发式增长。2023 年 MemGPT 提出"记忆即操作系统"的范式时，还几乎是独一份。短短两三年后的 2026 年初：

- **15+ 个专用记忆框架**（Mem0、Hindsight、OpenViking、Zep、Cognee、Letta、ByteRover、Supermemory、RetainDB、Honcho 等）
- **每家主要 AI 实验室**都在推出记忆功能（OpenAI ChatGPT Memory、Claude Memory、Gemini Personal Intelligence）
- **4+ 篇学术综述**提供分类体系和基准测试
- **活跃的基准测试竞争**，在 LongMemEval 和 LoCoMo 上的分数在 18 个月内从 60% 飙升到 96%

整个领域的核心议题已经从"能不能让智能体记住东西"变成了"怎么让它们记得*好*、大规模地记、保护隐私地记，同时不因过时信息而产生幻觉？"

本书后续章节将围绕这个问题展开深入探讨。
